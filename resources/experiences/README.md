# Development Experience Notes

開發經驗庫用來保存「當時怎麼做、為什麼這樣做、後來學到什麼」。它和 repo 清單不同：repo 清單追求好找，experience notes 追求保留脈絡與自然迭代。

## 設計原則

- 保留日期，不強迫改寫成永遠正確的結論。
- 記錄當時限制、替代方案與 trade-off。
- 允許後續追加 update notes，而不是覆蓋舊判斷。
- 把可複用的結論回流到 resource、case 或 skill。

## 建議檔名

```txt
YYYY-MM-DD-topic.md
```
